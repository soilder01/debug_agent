# 小D Debug Agent 企业级落地交付手册

## 文档定位
- 本文档主题：企业级交付。
- 主要读者：项目负责人、评测负责人、算法负责人、平台运维和验收人员。
- 文档用途：用于小D问答、项目验收、用户培训、运行排障和后续知识沉淀。
- 文档要求：每一条规则都应能被追溯到用户体验、工程状态、证据链或审计记录。

- 摘要：本文档定义小D Debug Agent作为部门级多模态调试平台的交付口径。
- 摘要：交付内容覆盖飞书入口、后端任务、证据链、报告、写回、审计、知识库和自优化。
- 摘要：本文档用于验收、培训、项目复盘和后续运营，不是简单功能列表。

## 01 背景和业务痛点
- 章节目标：围绕企业级交付说明背景和业务痛点在企业级落地中的具体要求。
- 适用读者：项目负责人、评测负责人、算法负责人、平台运维和验收人员。
- 交付约束：本章节内容必须能被小D问答、RAG检索或验收检查复用。
- 背景和业务痛点要求01：在“报告完成后的同步决策”场景中，系统必须把下一步动作落到可执行入口；依据优先查看`spreadsheet row mapping`，验收材料应包含`运行审计记录`，并在用户可见文案中说明边界。
- 背景和业务痛点要求02：在“证据耗尽后的人工复核”场景中，系统必须在卡片中暴露安全决策；依据优先查看`writeback audit`，验收材料应包含`RAG知识索引`，并在用户可见文案中说明边界。
- 背景和业务痛点要求03：在“上线前权限门禁检查”场景中，系统必须明确记录输入来源和处理范围；依据优先查看`final report document`，验收材料应包含`最终报告卡片`，并在用户可见文案中说明边界。
- 背景和业务痛点要求04：在“真实飞书群发起表格重跑”场景中，系统必须避免把候选判断写成确定结论；依据优先查看`operation audit`，验收材料应包含`飞书云文档报告`，并在用户可见文案中说明边界。
