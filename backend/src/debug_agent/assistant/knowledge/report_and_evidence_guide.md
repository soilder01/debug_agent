# 最终报告和证据链阅读指南

## 文档定位
- 本文档主题：报告和证据。
- 主要读者：需要判断结论可信度和后续动作的所有用户。
- 文档用途：用于小D问答、项目验收、用户培训、运行排障和后续知识沉淀。
- 文档要求：每一条规则都应能被追溯到用户体验、工程状态、证据链或审计记录。

- 摘要：本文档定义最终报告应该怎么读，以及证据链如何支撑结论。
- 摘要：报告必须同时说明结论、依据、边界和下一步。
- 摘要：没有supported causal comparison时，报告必须明确证据不足。
